# 課題作成ルール（Backlog / GitHub Issue）

Backlog 課題・GitHub Issue を作成する際は、以下のルールに必ず従うこと。

## 対象

- Backlog 課題の起票（`mcp__backlog__backlog_add_issue` 等）
- GitHub Issue の作成（`gh issue create` 等）

## 必須ルール

- テンプレートの全セクション（背景・目的 / 完了の定義 / 対応内容 / 対応範囲外 / 関連情報 / 補足 / 用語集）を使用する
- 空のセクションは削除せず `- なし` と明記する
- 完了の定義(DoD)は**検証可能な粒度**で書く
- 期限は**カレンダー上で特定できる日時・時期**で記載する（相対的な表現は禁止）
- 一般的でない用語がある場合は**用語集を必ず作成**する（GitHub: 脚注形式 / Backlog: リスト形式）

テンプレート本文と記載ガイドラインは references/issue-template.md を参照
